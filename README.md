# AuPromptKit — Australian Banking BA Toolkit

![License](https://img.shields.io/badge/license-ISC-blue.svg)
![Version](https://img.shields.io/badge/version-1.0.0-green.svg)

**AuPromptKit** is a specialized prompt library and utility suite designed for Business Analysts working within the **Australian Banking and Financial Services** sector. It provides high-quality, pre-tested prompts tailored for regulatory compliance (APRA, CDR, NPP), Agile delivery, and digital transformation.

## 🚀 Features

- **📖 Guidance & Knowledge Base**: Essential guides for Scrum facilitation, AU banking jargon, and app mastery.
- **🧠 User Story Intelligence**: An intelligent scanner that evaluates draft stories against **INVEST** standards and Australian banking compliance risks.
- **🪄 Feature Breakdown**: Generate comprehensive AI prompts to break down epics into detailed, estimated user stories.
- **📋 Prompt Library**: Hundreds of curated prompts for daily BA tasks, from tech spikes to stakeholder management.
- **🌓 Dark Mode**: A sleek, neo-corporate design for comfortable long-term use.

## 🛠️ Tech Stack

- **Frontend**: Vanilla HTML5, CSS3, and JavaScript (ES6+).
- **Backend**: Node.js with Express (for local serving and file intelligence).
- **Theming**: Premium neo-corporate design system with dark/light mode support.

## 📦 Getting Started

### Prerequisites

- [Node.js](https://nodejs.org/) (v14 or later recommended)

### Installation

1. Clone the repository:
   ```bash
   git clone https://github.com/vnkg113/au-banking-toolkit.git
   cd au-banking-toolkit
   ```

2. Install dependencies:
   ```bash
   npm install
   ```

3. Start the application:
   ```bash
   npm start
   ```

4. Open your browser at `http://localhost:8081`.

## 🛡️ Security & Privacy

This toolkit has been audited for privacy:
- **No PII**: No Personal Identifiable Information is stored or transmitted.
- **Generalized Data**: Personas and bank names are generalized for professional project use.
- **Local First**: All custom prompts are stored in your browser's `localStorage`.
